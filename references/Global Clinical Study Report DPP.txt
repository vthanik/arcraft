2 INTRODUCTION
2.1 Guidance for DPP Users and Reviewers
The Data Presentation Plan (DPP) contains comprehensive specifications for the development of data displays for an analysis.  
TLG Templates and Metadata Tables:
Within the DPP, the following TLG templates will reside:
* Global Standard Templates (GS): Common templates utilized by studies across therapeutic areas. 
* Therapeutic Standard Templates (TS): Common templates utilized by studies within a therapeutic area. 
* Compound Standard Templates (CS): Common templates utilized by studies within a compound.
* Study Specific Templates (SS): Custom templates constructed for the analysis. 

A Metadata Table will contain information on the exact layout for a table, listing or graph.

3.2 Conventions
Layout notes and programming notes can be specified in a specific metadata table or specified more broadly to apply across multiple tables, listings, graphs (TLGs) in this Conventions section. 
* Instructional text below provides guidance and global standard metadata options applying to several TLGs across domains or within domain 
* Default options are prepopulated in the metadata tables in Section 4
Other text below (e.g. subgroup, cutoff) will be retained in the final DPP if TLGs display such conventions 
3.2.1 General Conventions

Output File Name:
* Users must add output file name extension as specified in “11_05 - Managing GBDS Files - Formatting and Importing into PRISM Docbase” for all tables, listings and graphs created to support key business decisions, regulatory submissions, publications, and market access
* Users can specify other extensions (e.g., CSV) for other outputs such as CRF selection list for narrative, etc. 

Standard titles:
* For identifier options, the user will specify the identifier to describe the presentation, for example: 
* GS_CSR_AE_T_001, the standard title is Adverse Event Identifier Summary, for a Serious Adverse Event analysis, the Adverse Event Identifier will be Serious Adverse Event
* GS_CSR_DS_T_00x, the standard title is Milestone Identifiers Subject Status Summary, Milestone Identifiers could be End of Treatment Period, End of Follow Up, and End of Survival Follow Up
* GS_CSR_CM_T_00x, the standard title is Medication Category Identifier Summary, Medication Category Identifier could be Previous Medications, Current Medications, Concomitant Medications and Post Treatment Medications    
* When reporting by subgroup or spanning column (e.g. GS_CSR_AE_T_002), user-specified descriptive text for a specific type of subgroup/category will be appended to the title, e.g. Adverse Event Summary by Race.
* When the cutoff option is selected, “with at least X% cutoff” will be appended to the title, e.g. with at least 5% cutoff.

User can specify columns displayed:
* Groups and Total column (default in metadata table)
* Groups only
* Total column only

User can produce an output where data is presented separately for each value of the specified subgroup variable(s).

For outputs generated using SAS in RTF format: 
* Limit for number of footnote statements: Only include up to 8 footnote statements in the DPP metadata tables (Footnote 1 through Footnote 8). Footnotes beyond footnote 8 cannot be included in the metadata table.
* Footnote length: There is no limit on the length of each footnote. If a footnote exceeds the maximum line size of the output, it can be automatically wrapped onto the next line in the outputs. Manual breaking of footnotes is not needed in the DPP.

* Group = Treatment Group, unless otherwise specified in layout note.
Group represents study level defined grouping, e.g. treatments, cohorts

* Subgroup variables =XXX, unless otherwise specified in layout note. 

* When reporting enrolled subjects, re-enrolled subjects will be counted or displayed only once 

* Order of Summary Statistics in the table, unless otherwise specified in layout note.  
N
MEAN
SD (SE)
MEDIAN
Q1, Q3
MIN, MAX

* UNCODED is defined as adverse events or non-study medications (e.g. concomitant) which have not yet been coded and will be displayed as the last regardless of the sort order. Uncoded adverse events or medications will not occur in analyses like CSR. 

CRF collected date (e.g., in Adverse Events, Non-Study Medications, etc.) will be displayed in subject listings (i.e. not the imputed date), unless otherwise specified in layout notes

For categorical parameters that are collected from CRF, e.g. reasons for discontinuation of treatment, each category will only be displayed in summary analyses if an occurrence was reported (i.e. count greater than 0 in at least one group); For categorical parameters that are derived, e.g. age group “< 65” and “>= 65”, each category will be displayed in summary analyses even if no occurrence was reported (i.e. count equals 0). 

The purpose of the templates in this DPP is to give an idea of the output layout and presentation. The templates contain examples of data collected from CRF and is not intended to reflect the actual data collected for the study.
3.2.2 General AE Conventions
User-specified MedDRA dictionary displayed:
* SOC and PT (default in metadata table)
* SOC only
* PT only
* Customized AE group (e.g. cSMQ) and PT
* Customized AE group only

User-specified Row sorting: 
* SOC only or PT only
* Alphabetically
* Descending incidence*
* SOC and PT
* Alphabetically by SOC, alphabetically by PT 
* SOC descending incidence, PT descending incidence* (default in metadata table)
* Alphabetically by SOC, PT descending incidence*
* SOC descending incidence, alphabetically by PT*
* A secondary alphabetical sort will be applied in the case of ties in descending incidence

Cutoff is based on a percentage threshold on PTs which will display PTs >= the defined cutoff
* Cutoff is applied prior to rounding PT percentage to 1 decimal for display
* TOTAL SUBJECTS WITH AN EVENT will display all subjects with at least 1 event, not only subjects with at least 1 event that satisfied the cutoff criteria
* SOC/cSMQ shall not be displayed if no PTs within the SOC/cSMQ meet the criteria
* SOC/cSMQ row will display counts representing all subjects across all PTs within that SOC/cSMQ, not only subjects with events represented by PTs within that SOC/cSMQ that satisfied the cutoff criteria
* Cutoff applied to which group or category is specified in Programming Notes in Metadata Table 

3.2.3 General Disposition Conventions
* REASON criteria and NEXT EXPECTED STUDY PHASE shall 
* Table:
o Be listed in the order that they were displayed on the collection instrument 
Listing: 
o Reason criteria will be concatenated with additional details collected for “Specify” on the CRF using a semi-colon as the delimiter, e.g. “OTHER: PATIENTS WISH”
o Reason criteria will be concatenated with “COVID-19” using a front slash “/” as the delimiter to indicate reason criteria due to COVID-19 based on the CRF Covid-19 Yes/No field, e.g. ADVERSE EVENT/ COVID-19

3.2.4 General Laboratory Conventions
* Worst Toxicity Grade is defined as the worst post-baseline measurement (in one or both directions) for each respective lab test description in specified analysis period

* Marked Abnormality or Toxicity grade CATEGORIES  :
o Distinct or overlapping
o If include “Any Grade”, “Any Grade” includes missing grades

3.2.5 General Concomitant Medications Conventions
* For UMC WHO Drug Dictionary B3/C3 reporting, Generic Name (UMC WHO Drug standardized medication name) will report either:  
o a single ingredient generic (e.g., MORNIFLUMATE) 
o a multiple ingredient generic with each ingredient separated by a semi-colon (e.g., ALLANTOIN; BENZOCAINE; CAMPHOR; DIMETICONE; MENTHOL; WHITE SOFT PARAFFIN)
* For the UMC WHO Drug Dictionary ATC reporting:
o Subjects shall be counted only once at the anatomic class (UMC WHO Drug ATC1 text), and therapeutic class levels (UMC WHO Drug ATC2 text).; however, the generic name may appear in multiple therapeutic classes (e.g., ACETYLSALICYLIC ACID is in both ANTITHROMBOTIC AGENTS and ANALGESICS therapeutic classes)
o The counts and percentages of a generic name represent overall use, not use within the specifically noted therapeutic class
